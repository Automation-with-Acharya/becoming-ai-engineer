# Technical Questions Asked

## 1. Explain Monolithic vs Microservices Architecture

Initially, I took a few seconds to recall the term "Monolithic", but since I had recently studied Microservices, I explained the concept confidently.

### My Explanation

- Large applications should be divided into independent functional modules.
- Each module can be deployed separately.
- Failure of one module should not bring down the complete application.
- Easier maintenance.
- Better scalability.

Although I referred to them as **containers**, my overall explanation was technically correct.

### Examples I Gave

General Example

- Payment Module
- Login Module
- Form Submission Module

Company-Specific Example

Since the company develops Tender Management Portals:

- Document Upload Service
- Login & Authentication Service
- Tender Submission Service

The interviewer appeared satisfied and appreciated the practical examples.

# 4. Hashing vs Encoding

Question:

> Difference between Hashing and Encoding

My Answer:

Encoding

- Converts data into another format.
- Used for transmission or storage.
- Reversible.

Hashing

- Used for security.
- One-way transformation.
- Used to protect sensitive information.

---

# 5. Cyber Security

Question:

Explain

- Vulnerability
- Threat
- Risk

Instead of defining each individually, I explained using a practical example.

### Example Used

Suppose a web application has a Login Form.

Developer forgets SQL Injection protection.

This becomes a **Vulnerability**.

An attacker exploits this vulnerability.

This becomes the **Threat**.

The consequences include:

- System compromise
- Financial loss
- Reputation damage
- Operational impact

These consequences together become the **Risk**.

The interviewer was visibly satisfied, especially after I used **SQL Injection** as the practical example.

This was one of the strongest technical moments of the interview.
